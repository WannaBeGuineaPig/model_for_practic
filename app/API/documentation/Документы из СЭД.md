# Документы из СЭД

По кнопке *«Выгрузить данные из СЭД Лексемы»*  - собираются заявления от сотрудников, которые: 
* еще не были отправлены в 1С,<br> 
* прошли до конца по маршруту,  <br>
* и для этих подтипов в настроечной таблице указана Сущность 1с.<br>

По кнопке *«Загрузить данные в 1С»*  формируются файлы обмена согласно [«Настройкам параметров обмена с 1С»](../Настройка интеграции справочников и документов/Настройка интеграция справочников и документов/index.md) , подставляя  данные из строчек настройки по *реквизитам заявлений*.
![Рисунок 1 Вкладка_Исходящие_документы](./media/Вкладка_Исходящие_документы.png) <br>
 
	
Сущность файлов  при выгрузке данных преобразуется из двоичного формата в формат 64base. <br>
Отправка осуществляется по 1 строке, в ответ возвращается код документа.<br>
По этому ответу  в таблице соответствия формируется строка:<br>
*Код филиала, Таблица СЭД, Код 1С, Тип и номер 1С, Код документа Лексемы, Подтип документа Лексемы*.<br>
После появления данных в 1С, например, для документа «Отпуск», в карточке документа указываются недостающие данные, производится расчет, вкладывается печатная форма.<br>
При *появлении **нового типа** заявления*, обмен которыми необходим между системами, нужно внести новую строчку и заполнить все необходимые поля.   <br>
Для заполнения  поля *«Справочник в Лексеме»* необходимо сначала внести новый подтип заявления в «Конструктор документов».<br>
Также нужно заполнить по новому типу документа (сущности 1С) необходимые строчки, например, для приказа на отпуск, это:<br>
•	Заявления от сотрудника<br>
•	Реквизиты заявлений от сотрудника<br>

Поле «Наименование документа» должно быть одинаковым по всем строчкам, относящимся к одной сущности на закладке «Исходящие документы».<br>
![Рисунок 2 Наименование_документа](./media/Наименование_документа.png) <br>

на вкладке «Входящие документы» заполняем:

•	Подтип документа на подписание
![Рисунок 3 Подтип_отпуск](./media/Подтип_отпуск.png) <br>
