# Заявка на потребность 

**Заявка на потребность** –  это заявка в производство (ТМЦ может быть в наличии на складе), а также на приобретение товаров, работ, услуг. Инициатором потребности может выступать как рядовой сотрудник (например, заявка на приобретение оборудования для выполнения своей работы), так и руководитель, его заместитель, либо специально выделенный сотрудник в отделе (например, формировать потребности для своего подразделения).

Существует 2 способа создания Заявки на потребность:

1. создание документа из реестра Заявок на потребность;

2. создание документа на основании Дефектной ведомости. 

## Способ 1. Создание Заявки на потребность с 0.

Для работы с документом откройте реестр Заявок на потребность. Для создания нового документа нажмите кнопку **Создать**.

![](image.png)

Поля, которые заполняются по умолчанию: 

* **Номер** – заполняется автоматически при сохранении документа. Возможности редактировать нет;

* **Дата документа** - по умолчанию заполняется текущей датой;

* **Инициатор** - заполняется сотрудником, создающим документ;

* **Подразделение** - подтягивается подразделение инициатора;

* **Статус** - автоматически заполняется значением “Новая”. При повторном открытии статус будет "Создана"

![](image-1.png)

* **Вид потребности, Проект, Вид, Основание** - необязательные для заполнения поля. Данные вносятся путем выбора из выпадающего списка, который открывается нажатием на ![](4.png)

![](image-2.png)

Для заполнения табличной части нажмите на кнопку Добавить, чтобы появилась новая строка. Автоматически заполнятся поля **Проект** из шапки документа, а также **Статус**. 

![](image-3.png)

Если в дальнейшем нужно будет поменять проект, то появится всплывающее окно с вопросом о замене проекта в табличной части.

![](image-4.png)

В колонке Код/наименование материала выберите необходимую позицию из выпадающего списка. Автоматически заполнятся такие данные как: Группа и код номенклатуры, Ед.измерения.

![](image-5.png)

**Укажите Дату потребности** - это обязательное поле при заполнении табличной части. Без даты система не даст сохранить документ. 

![](image-6.png)

!!! note "Примечание"
    Обратите внимание, что Дата потребности не может быть раньше даты создания документа. 

![](9.png)

Укажите Количество Затребовано, а также Цену номенклатуры. Автоматически рассчитается Количество К закупу и Сумма. 

![](image-7.png)

По такому принципу можно добавить любое количество позиций номенклатуры. 

В колонке Замена указывается номенклатура, которая придет на замену той, что указывается в колонке Код/наименование материала.

![](10.png)

**В Заявке на потребность есть 2 способа создания позиций, которых нет в справочнике Номенклатур:**

1) ***Создание номенклатуры по кнопке ![](26.png).***

Благодаря данной кнопке есть возможность одновременно сохранить сразу несколько позиций номенклатуры ручного ввода в справочник.

Шаги, благодаря которым произойдет успешное создание номенклатуры:

- ввести наименования в колонку Номенклатура(ручной ввод);

- указать Группу номенклатуры;
  
- укажить Ед.измерения;

- одновременно зажать клавишу CTRL + левую кнопку мыши на нужных строках с материалами, чтобы выделить их;

- затем нажать на кнопку Создать номенклатуру.

![](11.png)

После завершения операции появится соответствующее уведомление и номенклатура и номенклатурный код появятся в колонке Код/наименование материала:

![](12.png)

Укажите Даты потребности, Количества и Цены, чтобы сохранить документ. 

![](13.png)

2) ***Создание номенклатуры по кнопке ![](27.png).***

Благодаря данной кнопке есть возможность создать 1 вид номенклатуры, не открывая справочник Номенклатур. 

Добавьте новую строку и нажмите на кнопку ![](27-1.png) в поле Код/наименование материала. 

![](28.png)

Откроется окно ввода новой номенклатуры, в котором требуется заполнить следующие поля:

* **Наименование краткое** – текстовое поле, обязательное для заполнения;

* **Наименование полное** - текстовое поле , обязательное для заполнения. Оно автоматически заполняется значением поля “Наименование краткое”;

* **Базовая ед.измерения** – поле, обязательное для заполнения. Значение выбирается из выпадающего списка;

* **Группа номенклатуры** – поле, обязательное для заполнения. Значение выбирается из выпадающего списка или обозревателя аналитик (если вы уже создали в этом документе номенклатуру, то при создании последующих это поле заполнится автоматически исходя из раннее созданной вами номенклатуры). Поле можно редактировать;

* **Вид номенклатуры** – поле, обязательное для заполнения. Проставляется автоматически, но поле можно редактировать. 

![](29.png)

Остальные поля не обязательны для заполнения, данные в них вносятся по желанию. Сохраните документ, номенклатура появится в поле “Код/наименование материала”, подтянется Ед.измерения и Группа номенклатуры.

![](30.png)

## Способ 2. Создание документа на основании Дефектной ведомости.

**Создать Заявку на потребность на основании Дефектной ведомости возможно двумя методами:**

1. В Заявке на потребность нажмите на Операции – Загрузить дефектные ведомости;

2. В Дефектной ведомости нажмите на Операции – Создать Заявку на потребность.

Рассмотрим первый метод:

Для работы с документом откройте реестр Заявок на потребность. Для создания нового документа нажмите кнопку **Создать**.

![](image-8.png)

В открывшемся новом документе нажмите на Операции – Загрузить дефектные ведомости:

![](image-9.png)

Откроется окно, в котором необходимо указать Период, за который были созданы Дефектные ведомости, после этого нажмите кнопку Загрузить. 

![](image-10.png)

Подтянутся все Дефектные ведомости за указанный период. Выделите необходимые позиции, а затем нажмите Вставить выбранные. 

![](image-11.png)

В табличной части Заявки на потребность отобразятся позиции из Дефектной ведомости:

![](image-12.png)

Запоните обязательные поля и сохраните документ по кнопке ![](image-13.png)

# Согласование заявок на потребность

 Данная форма собирает для пользователя все заявки на потребность, которые находятся у него на текущем этапе на маршруте с действием «согласовать». Пользователь, может не открывая каждую потребность согласовать все одновременно, либо выделить и согласовать конкретные, также может вносить правки, которые будут указаны в соответствующей заявке на потребность. Для этого нужна роль **RequestPurchaseAgreement_write**.

 Форму можно найти в главном меню системы, она находится в модуле "Управление закупками".

 ![](image-20.png)

 После нажатия на **Согласование заявок на потребность** перед вами откроется форма всех заявок на потребность, которые находятся у вас на этапе согласования по маршруту.

 ![](image-21.png)

 Двойным щелчком по заявке на потребность можно перейти в нее. Она откроется в отдельной вкладке.

 ![](image-22.png)

 Для того, что бы согласовать разом все или несколько заявок на потребность, вам потребуется отметить их галочкой и нажать согласовать.

 ![](image-23.png)

 После этого перед вами появится окно, в котором будет указано количество заявок н апотребность для согласования, нажмите **"Да"**.

 ![](image-24.png)

 Перед вами появится окно с завершением операции и документ отправится дальше по маршруту.
