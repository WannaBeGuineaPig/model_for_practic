# Локальные нормативные акты

## Общее описание назначения документа

Документ "Локальные нормативные акты" (далее ЛНА) предназначен для создания документов, с которыми требуется ознакомить сотрудников предприятия. Например, к данной группе относятся: Карта процессов, Методические руководства, Извещения об изменениях и т.д.

<!-- ![type:video](https://www.youtube.com/embed/Em3GlAkrczE) -->

Существует несколько вариантов создания ЛНА:

* **ЛНА на бумажном носителе** - данный документ сотрудники подписывают на бумажном носителе при приеме на работу. В данном случае создается ЛНА, проставляется признак **"Ознакомлен ранее"** во вкладке ["Распространяется на"](#вкладка_распространяется_на), далее необходимо через операции выбрать **"Разместить в личном кабинете"** и данный ЛНА автоматически разместится в [личном кабинете сотрудника](../Профиль сотрудника/#нормативные_акты).

* **ЛНА в электронном виде** - данный документ является электронным. Он [создается](#документ) в системе после этого направляется для ознакомления работающим сотрудникам. Далее по задаче планировщика данный ЛНА будет автоматически направляться вновь принятым сотрудникам для ознакомления.

* **ЛНА на бумажном носителе переведенный в электронный** - данный документ был ранее подписан на бумажном носителе , но был переведен в электронный формат. В данном случае такой документ сперва заводится в системе с признаком **Ознакомлен ранее**, направляется в личный кабинет сотрудника, после этого признак ознакомления снимается. Такой документ также будет по задаче планировщика автоматически направляться вновь принятым сотрудникам для ознакомления.

## Реестр

Для открытия реестра документов необходимо перейти в Меню - Кадровый документооборот - Локальные нормативные акты.

<!-- Либо через пункт плиточного меню, если он присутствует

![Плиточное меню](media/tiled.png) -->

Откроется реестр документов. 

![Реестр "Локальные нормативные акты"](media/registry.png)

- **Статус ознакомления** - отображается статус ознакомления с ЛНА текущего пользователя. Если пользователь уже ознакомился с документом, то в ячейке содержится значение "Подписан" с зелёным значком. В случае, если еще не ознакомился в ячейке содержится значение "Не подписан" без значка;
- **Тип** - отображается тип документа;
- **Номер** - отображается номер документа;
- **Краткое содержание** - отображается краткое содержание документа;
- **Организация** - отображается организация, на которую распространяется документ;
- **Текст** - отображается текст документа;
- **Статус** - отображается статус документа (утвержден или проект);
- **Зона распространения** - отображаются подразделения, должности и/или сотрудники на которых распространяется действие документа;
- **Маршрут** - отображается вся информация по маршруту (не запущен, текущий, завершен, отменен);
- В столбце **Шаблон** отображается по какому шаблону был создан маршрут;
- **Удален** - отображает, является ли данный документ удаленным или архивным. Такие документы могут видеть только пользователи с определенными правами;
- **Ознакомлен ранее** - признак проставляется у документов, которые доступны сотруднику, но не требуют ознакомления в системе. Используется для ЛНА, которые сотрудники подписывают на бумажном носителе при приеме на работу.
- **"Подписан КЭП"** - отображается зеленый значок "Подписан КЭП" в тех локальных нормативных актах, в которых в завершенном маршруте подписания присутствует как минимум одна [квалифицированная электронная подпись](../../ЭП/#квалифицированная_электронная_подпись_или_кэп)(данная настройка подключается администратором);
- **Переподписание документа КЭП** - отображает, был ли данный документ [переподписан](../../Руководство для руководителя/Переподписание КЭП/) с помощью квалифицированной электронной подписи;

## Документ

Создавать новый локальный нормативный акт может пользователь, у которого назначены специальные права. Для создания нового документа необходимо нажать на кнопку ![создать](media/image3.png). 

Новый документ выглядит следующим образом: 

![Документ "Локальный нормативные акты"](media/image4.png)

### Вкладка "Основная"

Для создания нового документа необходимо вначале выбрать из справочника Подтип документа.

![Выбор подтипа ЛНА](media/subtypeChoose.png)

После выбора нужного подтипа документа на форме появляются новые параметры "Реквизиты документа", а также формируется часть для автозаполнения в текстовом поле.

![Результат выбора шаблона документа](media/image6.png)

В документ необходимо добавить вложение для ознакомления. Без добавленного вложения документ не сохранится в [системе](#дальнейшая_работа_с_документом). 

На вкладке "Основная" даты в поле "Действует с/по" заполняются в случае если у ЛНА присутствует срок действия. В остальных случаях проставлять даты не нужно.

В случае если ЛНА утвержден приказом по предприятию, необходимо указать номер и дату приказа.

Также необходимо заполнить "Реквизиты документа":

![Пример заполнения реквизитов документа (атрибутов)](media/attributesFilledP.png)

После заполнения реквизитов необходимо нажать на кнопку "Заполнить шаблон". После нажатия на кнопку данные реквизиты в качестве параметров передаются в текстовую часть документа и заменяют прописанные там входные параметры, заключенные в '#'.

![Пример заполнения шаблона введенными реквезитами](media/templateFilledP.png)

<!-- При нажатии на кнопку "Заполнить шаблон" документ сохраняется до тех пор, пока его не закроют (т.е. можно использовать кнопку "Обновить" на форме без потери документа). ***Чтобы документ сохранился, требуется наличие в нем вложений.*** -->

### Вкладка "Распространяется на"

Далее необходимо перейти к заполнению второй вкладки формы: **Распространяется на**.

На этой вкладке ответственные исполнители указывают список подразделений и/или должностей, либо сотрудников, на которых распространяет свое действие создаваемый нормативный документ. Исходя из заданных таблиц, она определяет итоговый набор сотрудников, которым данный документ должен прийти для ознакомления.

![Вкладка "Распространяется на"](media/LNA-1.png)


Для быстрого заполнения вкладки можно выбрать одну из групп рассылки, которые определяются в реестре [Группы рассылки](../../Группы рассылки/)

После выбора группы рассылки из справочника подтягиваются данные по Подразделениям, Должностям и Сотрудникам.

![Вкладка "Распространяется на"](media/LNA-2.png)

Для внесения в список рассылки подразделения, должности или сотрудника нужно нажать на кнопку "+" в соответствующей таблице. Также в табличной части возможно добавить сотрудников, которых необходимо исключить из списка рассылки. Если в [группе рассылки](../../Группы рассылки/) присутствуют пользователи, они также автоматически добавляются в исключаемую таблицу.

![Пример заполнения вкладки "Распространяется на"](media/image5.png)

![Направления](media/zone.png)

Существует несколько способов формирования списка рассылки, которые задаются при помощи поля «Выбрать способ формирования рассылки»:

**1 способ** - _Сотрудник относится к указанному подразделению ИЛИ имеет указанную должность._ В данном примере локальный нормативный акт будет направлен на ознакомление всем сотрудникам, которые работают в подразделении _"Администрация УК"_, а также всем сотрудникам имеющим должность _"Ведущий инженер"_ вне зависимости от того, в каком подразделении они работают.

![Рассылка](media/docForReadList.png)

![Рассылка администрации](media/adminPod.png)

**2 способ** - _Сотрудник относится к указанному подразделению И имеет указанную должность._ В данном примере локальный нормативный акт будет направлен на ознакомление только сотрудникам, которые работают в подразделении _"Администрация УК"_ и имеют должность _"Ведущий инженер"_.

![Должность](media/dolnjnost.png)

![Должности](media/dolnjosti.png)

**3 способ** - В случае если необходимо отправить локальный нормативный акт на ознакомление **только определенным сотрудникам** без указания подразделения и/или должности, необходимо очистить поле "Выбрать способ формирования списка рассылки". В данном примере локальный нормативный акт будет направлен на ознакомление только сотрудникам Денежной Е.К. и Антипову В.А.

![Способ](media/LNA-3.png)

![Сотрудники](media/rassilka12.png)

Для третьего способа в системе есть возможность загрузки списка с сотрудниками в систему, которые автоматически будут добавлены в таблицу **"Сотрудники"**. Для этого необходимо в панели инструментов нажать на кнопку **"Операции"** и выбрать **"Сформировать шаблон для загрузки сотрудников"**.

![Сформировать](media/sformirovat.png)

Автоматически будет скачан документ в формате xlsx, который необходимо будет заполнить Ф.И.О. и табельный номер сотрудников и сохранить файл.

![xlsx](media/xlsx.png)

Для загрузки списка в панели инструментов необходимо в операциях нажать на **"Загрузить сотрудников"** и выбрать сформированный список.

![Загрузка](media/load.png)

После загрузки файла через некоторое время сотрудники будут автоматически добавлены в таблицу.

![Загруженные сотрудники](media/loadSot.png)

Также система автоматически скачает файл в формате xlsx, в котором будут указаны сотрудники, которые не были добавлены в рассылку.

![Незагруженные](media/noLoad.png)

В ЛНА имеется строка **"Исключить из листа ознакомления подписавших ЛНА при помощи ЭП"** с признаком, который при отмеченном признаке позволяет исключать из списока рассылки сотрудников учавствующих в маршруте документа с дейсвием "Подписать ЭП" и "Подписать КЭП".

![Рисунок](media/LNA-4.png)

Поле **Срок ознакомления с нормативным документом** - необязательно для заполнения. Если проставить дату, то срок на ознакомление будет отчитываться от нее.

**Ознакомлен ранее** - признак проставляется у документов, которые доступны сотруднику, но не требуют ознакомления в системе. Используется для ЛНА, которые сотрудники подписывают на бумажном носителе при приеме на работу. Чтобы такие документы отображались в [личном кабинете сотрудника](../../Руководство линейного сотрудника/Профиль сотрудника/#нормативные_акты), необходимо в панели инструментов нажать на кнопку "Операции" (данная кнопка появляется после утверждения ЛНА), и выбрать **"Разместить в личном кабинете"**.

![Отправка в кабинет](media/toCabinet.png)

При нажатии на **"Утвердить и отправить на ознакомление"** локальныйнормативный акт будет автоматически утвержден и отправлен на ознакомление согласно списка адресатов. После утверждения появится всплывающее окно, в котором будет предложено просмотреть [предварительный список адресатов](#предварительный_список_адресатов). Если нажать "Да" в новом окне отобразится предварительный список адресатов. Если непросматривать, то ЛНА будет утвержден и отправлен на ознакомление согласно зоны распространения.

![Конфирм](media/confirm.png)

### Предварительный список адресатов

В системе есть возможность просмотра списка адресатов, которым будет направлен на ознакомление данный локальный нормативный акт. В этот список попадают сотрудники, которые соответствуют условиям [распространения документа](#вкладка_распространяется_на). Чтобы просмотреть список адресатов необходимо определить список подразделений и/или должностей, либо сотрудников, на которых распространяет свое действие создаваемый нормативный документ, сохранить локальный нормативный акт, нажать на кнопку "Операции" и выбрать "Предварительный список адресатов".

![Операции](media/LNA-5.png)

Откроется всплывающее окно со списком адресатом:

![Список](media/list.png)

В списке отображается следующая информация:

* Ф.И.О. получателя;
* Табельный номер;
* Подразделение работника;
* Родительское подразделение;
* Должность работника;
* Каналы оповещения - в случае если у пользователя настроено получение уведомлений через каналы [оповещений SMS и телеграмм](../Управление пользователями/index.md#создание_пользователя)

!!! Примечание
    В случае если сотрудник является совместителем, то в предварительном списке будет отображаться основное место работы.

При нажатии на "Утвердить и отправить на ознакомление" локальный нормативный акт будет автоматически утвержден и отправлен на ознакомление согласно списка адресатов.

**Дополнительные каналы уведомлений** - при включенных опциях сотруднику будет дополнительно направлены уведомления об необходимости ознакомления по каналам связи телеграм и смс, если они настроены.

Существуют два варианта дальнейшей работы с данным документом:

- 1 вариант: ответственный сотрудник [вкладывает](#добавление_вложений_в_документ) скан-копию ранее утвержденного ЛНА, например, "Положение о премировании сотрудников", переводит его в статус ["Утвержден"](#утверждение_документа) и направляет на [ознакомление](#отправка_документа_на_ознакомление) пользователям.

- 2 вариант: документ проходит стадию согласования по [маршруту](../../Работа с маршрутами/) у руководителя, [утверждается](#утверждение_документа) и после этого направляется [ознакомление](#отправка_документа_на_ознакомление) пользователям.

### Добавление вложений в документ

Для добавления вложения к документу необходимо нажать на соответствующую кнопку в панели инструментов, и выбрать необходимый документ, который будет загружен. После добавления документ будет отображен в окне вложений к документу.

![alt text](media/image-7.png)

Подробнее про работу со вложениями можно почитать в разделе ["Работа со вложениями"](../Работа со вложениями/index.md).

!!! note
    Для корректного отображения файла в предпросмотре, рекомендуется вкладывать файл в формате PDF.

### Утверждение документа

Чтобы утвердить документ, необходимо перевести его в статус "Утверждён" при помощи соответствующей кнопки во вкладке "Основная".

![Утвержден](media/changeStatusToDone.png)

При отсутствии вложений к документу система выдаст соответствующее оповещение:

![Уведомление](media/push2.png)

### Согласование ЛНА по маршруту

Для согласования документа необходимо отправить его по маршруту, нажав на кнопку !["Маршрут"](media/image9.png)

После нажатия на кнопку "Маршрут" необходимо нажать на кнопку создать "Создать маршрут". После этапа согласования локального нормативного акта должен присутствовать этап с ответственным сотрудником, который направит документ на ознакомление. Более подробно про работу с маршрутами можно прочесть в статье [Как отредактировать маршрут?](../../Работа с маршрутами/).

![Маршрут](media/route.png)

!!! warning

    После запуска маршрута редактировать документ и добавлять вложения запрещается.

Если документ не имеет вложений, система сообщит об этом при отправке документа по маршруту.

![Нет вложения](media/netVlozhenia.png)

### Отправка документа на ознакомление

Для формирования списка рассылки по заполненным на вкладке "Распространяется на" таблицам необходимо, чтобы статус документа был "Утвержден" и к документу был приложен утверждённый/подписанный ЛНА.

В утверждённом документе на панели инструментов появится кнопка "Операции" с пунктом "Отправить на ознакомление".

![Кнопка "Операции"](media/createReadList.png)

В системе появится уведомление что данный ЛНА был отправлен на ознакомление

![ЛНА](media/lndOtpravleno.png)

По нажатию на пункт "Отправить на ознакомление" локальный нормативный акт будет направлен пользователям по списку рассылки. Для просмотра списка пользователей, которым был направлен документ необходимо нажать на кнопку "Список рассылки".

![Документ для ознакомления](media/docForReadList.png)

Более подробно работа со списками рассылки описана в разделе [Списки рассылки](../../Обработка документов/#список_рассылки).

Как ознакомиться с локальным нормативным актом можно почитать в разделе ["Обработка документов"](../../Обработка документов/).

В случае если пользователь не успел вовремя ознакомиться с документом до окончания даты действия (если такие указаны во вкладке ["Основная"](#вкладка_основная)), то с помощью функции планировщика задач, система автоматически снимет с ознакомления таких пользователей.

#### Переслать документ

С помощью данной операции,например, руководитель подразделения может дополнительно ознакомить своих подчиненных с локальным нормативным актом по предприятию, в случае если они не попали под список распространения. При нажатии открывается всплывающее окно.

![alt text](media/image-2.png)

* Выберите сотрудника - выпадающий список сотрудников предприятия.

![alt text](media/image-3.png)

* Выберите шаблон рассылки - выпадающий список [пользовательской группы рассылки](../Пользовательские списки рассылки/index.md). Пользователю показываются только те группы владельцев где он является владельцем.

![alt text](media/image-4.png)

После выбора в таблице автоматически добавятся сотрудники

![alt text](media/image-5.png)

После этого необходимо нажать на кнопку "Отправить на ознакомление". Выбраннам сотрудникам автоматически будет направлен данный ЛНА на ознакомление.

![alt text](media/image-6.png)

!!! note 
    Сотрудники могут быть не добавлены по следующим причинам:
    - Сотрудник не является пользователем системы;
    - Не проставлена дата перехода КЭДО;
    - Сотрудник добавлен в список исключения в [группе рассылки](../Группы рассылки/)
