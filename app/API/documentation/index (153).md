# Исходящие документы

Документооборот **исходящих документов** - это оформление документов (писем), которые написаны в индивидуальном порядке или в ответ на входящий документ. Адресатами могут быть как частные физические лица, так и другие компании. Все виды исходящих документов оформляются на официальном бланке организации и подлежат обязательной регистрации. Как правило, исходящие документы готовят специалисты или руководители подразделений.

Процесс обработки исходящего документа в системе может содержать этапы Разработка проекта, Согласование, Утверждение (Подписание), Подготовка к подписанию (применимо при подписании бумажного документа), Регистрация, Информирование (Ознакомление), Отправка

## Работа с документом

![alt text](media/scr1001.png)

**Инициатор**

1. Создает документ. Откроется форма документа «Исходящий документ»
   
Если исходящий документ оформляется без привязки ко входящему документу, то нужно выбрать в главном меню пункт **«Исходящие документы»**.

Откроется реестр исходящих документов за выбранный период. Кнопка «Создать» позволяет сформировать новую форму для регистрации документа.

![alt text](media/image-6.png)

1. Заполняет обязательные поля документа . Исходящий номер присвоится при регистрации документа.

2. Прикрепляет проект исходящего документа по кнопке «Вложения» на панели инструментов

![alt text](media/image-1.png)

4. Создает маршрут из шаблона по кнопке «Маршрут» на панели инструментов. Если нужно отредактировать список согласующих, то вносит или удаляет их вручную с помощью кнопок управления маршрутом («Этап, участник», «Участник», «Удалить»). Более подробно работа с маршрутами описана в разделе [Как отредактировать маршрут?](../../Работа с маршрутами/).

![alt text](media/image-3.png)

5. По кнопке «Отправить» в маршруте документа отправляет проект документа на согласование.

**Согласующее лицо** (может быть несколько) - участник маршрута с видом действия «Согласовать». Переходит в проект исходящего документа, проверяет и отправляет далее по маршруту.

![alt text](media/2024-12-17_10-33-01.png)

**Регистратор** с видом действия «**Зарегистрировать»**.

1.  Регистрирует документ, нажав кнопку **«Зарегистрировать»,** при этом документу присваивается номер и фиксируется дата регистрации.

![alt text](media/image-4.png)

2.  Отправляет письмо на электронную почту.

3.  Нажимает кнопку «**Отправить»** в маршруте документа, при этом если есть список на ознакомление, то документ поступает соответствующим лицам, в противном случае маршрут завершается.



## Вкладка "Основная"

**Для исходящего документа обычно обязательными полями являются:**

* Тип документа;
* Рег.данные - номер/от;
* Дата документа;
* На чье имя;
* Краткое содержание;
* Содержательная часть.

Так же при необходимости в карточке _Исходящего документа_ имеется "Кнопка создания" напротив поля _"Контрагент"_ которая позволяет создать новую карточку контрагента. Подробнее ознакомиться с наглядным примером можно на скринах ниже:

**Реквизиты исходящего документа**:

"Тип документа" - тип исходящего документа, выбирается из справочника подтипов документов. Права на заполнение справочника есть у Администратора системы. Могут быть следующие типы:

    a)	Выписка
    b)	Заявка
    c)	Заявление
    d)	Письмо
    e)	Повестка
    f)	Постановление/приказ
    g)	Сопроводительное письмо
    h)	Судебное письмо
    i)	Телефонограмма
    j)	Тендерное предложение
    k)	Уведомление
    l)	Претензия
    m)	Извещение
    n)	Определение

Рег. № документа/от | Строка, Дата | Запоняется как вручную так или автоматом при нажатии на кнопку "Зарегистрировать" находящуюся рядом.
В ответ на входящий номер/от | Строка, Дата | Заполняется в ручную, присваивается номер для данного документа, который будет указан в исходящем документе, если создать исходящий документ на основе данного документа через кнопку в панели управления "Операции".
**От кого** |  |

Должность | Справочник | Должность составителя документа. Заполняется автоматически.
Подразделение | Справочник | Подразделение составителя документа. Заполняется автоматически.
Организация | Справочник |  Организация составителя документа. Заполняется автоматически.
Подписант (Ф.И.О.) | Справочник | Заполняется ФИО руководителя инициатора (составителя) документа. Если документ создает руководитель подразделения, то в данном поле будет указан инициатор.

Контрагент (текст) | Строка | Текстовое значение в котором указывается организация, которая отсутствует в справочнике контрагентов.

Копия (электронная почта) | Строка |  Копия электронной почты кому необходимо дополнительно направить документ кроме основного получателя, Эл. Почта вводиться как вручную так возможно выбрать из всплывающего окна по кнопку +Получатель
Телефон | Строка | Контактный электронный адрес организации-адресата, заполняется так же вручную или по примеру e-mail, необходимо в контрагенте во вкладке "контакты и адреса" указать в поле "Телефон" номер.
Краткое содержание | Строка | Краткое содержание документа, заполняется вручную.

Примечание | Строка | Свободный текст резолюции, заполняется вручную.


## Вкладка "Дополнительная"

Название реквизита | Тип реквизита | Описание
-------------------|---------------|------------
Журнал регистрации | Справочник | При создании документа пользователем журнал определяется автоматически в зависимости от типа документа и подразделения пользователя. Не может быть изменено. Настройка журналов регистрации происходит в документе [Журнал регистрации](../Журнал регистрации/index.md).
С уведомлением | Флажок | Признак, что письмо доставлено с уведомлением, вводится вручную.
Информация об уведомлении | Строка | Дополнительные комментарии к уведомлению. Вводится вручную.
Срочность | Справочник | Срочность документа. Для данного поля должны быть настроены соответствующий справочник в системе и шаблон маршрута в зависимости от вида срочности.
Требуется ответ | Флажок| Ставится признак, если обязательно ответное письмо. Заполняется вручную.
Срок предоставления ответа | Дата | Заполняется вручную, если есть признак "Требуется ответ". В системе можно настроить автоматическое проставление даты.
Ответ предоставлен в срок | Дата | Заполняется вручную после получения ответного документа.

"**Отправить адресату**" - используется , если документ является исходящим из одной организации и одновременно входящим для другой организации внутри холдинга. Кнопка активна на этапе регистрации документа. При нажатии на кнопку "История отправлений", на панели инструментов, откроется всплывающее окно с историей отправлений данного исходящего письма.

## Копирование исходящего документа

Если необходимо скопировать исходящий документ, необходимо выбрать нужный в реестре и нажать кнопку "Копировать". Система запросит подтверждение копирования документа. После согласия на копирование система автоматически откроет скопированный документ, который при необходимости возможно отредактировать, и после этого запустить вновь по маршруту. Скопировать можно исходящий документ как с завершенным, так и не завершенным маршрутом согласования.
