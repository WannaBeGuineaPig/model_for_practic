# Договоры

## Общее значение документа

**Договор** – соглашение двух или нескольких лиц об установлении, изменении или прекращении взаимных прав или обязанностей. 

Правильно организованная, слаженная работа с договорами и автоматизация процесса согласования практически полностью исключает риски нарушения сроков заключения договора, а сам процесс делает прозрачным.

**Согласование договора ведется следующим образом:**

1. подготовление проекта договора инициатором;
   
2. согласование договора с заинтересованными подразделениями (лицами);
   
3. подписание договора генеральным директором или иным уполномоченным доверенным лицом;
   
4. подписание договора контрагентом; 
   
5. направление подразделением-инициатором контрольных экземпляров договорных документов (с обязательным приложением спецификации к ним) в юридический отдел для регистрации;
   
6.  регистрирование договорных документов и присвоение регистрационного номера юридическим отделом.

### Вкладка “Основная”

Данная вкладка поделена на несколько окон для внесения информации:

#### 1. Окно "Регистрация договора". В ней отображаются регистрационные данные договора.

![](image.png)

* **Полный номер** – присваивается каждому договору, дополнительному документу. До регистрации договора поле “полный номер” можно оставить пустым, но во время нажатия на кнопку “Зарегистрировать” оно должно быть заполнено необходимыми данными.

* **Внутренний номер** – используется для нумерации договоров внутри системы. 

* **Входящий номер** – поле, данные которого берутся из документа, присланного заказчиком.

1. Номера договору присваивает юрист при регистрации.
Если поля “полный номер”, “внутренний номер”, “входящий номер” пусты, то при нажатии юристом кнопки “Зарегистрировать” они заполняются данными автоматически.

***Пример:***

1. Если задать только **внутренний номер**, то при регистрации договора система заполнит данные поля “полный номер” автоматически. Полный номер будет иметь вид “текущий год - внутр.номер”.

***Пример:***

![](image-11.png)

1. Если задать только **входящий номер**, то при регистрации договора система заполнит данные полей “полный номер” и “внутренний номер” автоматически. "Полный номер" и "внутренний номер" будут идентичны друг другу. 

***Пример:***

![](image-12.png)

* **Датой регистрации договора** является дата предоставления договора подразделением-исполнителем в юридический отдел для регистрации. 

* **Дата договора** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Система не даст сохранить документ, если оно останется пустым. 

* **Действует с** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Заполняется системой автоматически при создании документа. По желанию, начало действия договора можно установить свое. Для этого потребуется прописать дату вручную, либо выбрать ее по клику на ![](7.jpg).

* **Действует по** – дата окончания действия договора. Данное поле может остаться незаполненным, если проставлен признак “Автопролонгация” и выбрана дата автопролонгации.
 
***Примечание:*** В случае, если признак “Автопролонгация” не проставлен и статус договора = “Утвержден(Подписан)”, “На согласовании”, “Отправлен контрагенту”, “Заключен” поле “Действует по” обязательно для заполнения. 

* **Статус** - заполняется в зависимости прохождения маршрута.(настроить маршрут можно в настройке документа)
  
* **Кнопка “Зарегистрировать”** служит для регистрации договора. Для того, чтобы произвести регистрацию, потребуется заполнить обязательные поля, такие поля как: ***“Дата договора”, “Контрагент 1”, “Вид договора”,“Тип догвора”***. 

* **Оригинал представлен** проставляется после согласования проекта договора. Куратор прикладывает оригинал документа и ставит этот признак.

* **Уведомлять при истечении срока действия** – это значит, что уведомление уходит куратору договора по истечении срока действия. Галочку можно поставить только после заполнения обязательных полей и указания конечной даты действия договора.

#### 2. Окно "Сведения о контрагентах".

![](image-13.png)

* **Контрагент 1** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Поиск из справочника ведется во второй ячейке по ИНН, краткому или полному наименованию. Если известен код контрагента в системе, можно ввести в первой ячейке. Так же есть возможность создать нового контрагента или выбрать его из обозревателя аналитик с помощью кнопок ![](image-15.png)

![](image-2.png)

Выбрав таким способом Контрагента, автоматически заполнятся поля ИНН, КПП.

![](image-14.png)

* **Контрагент 2** – поле, которое не обязательно для заполнения. Данные вносятся по аналогии с “Контрагент 1*”. ИНН, КПП заполнятся автоматически. 

* **Тип договора** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Данные вносятся путем выбора позиции из списка.

![](image-16.png)

* **Вид договора** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Данные вносятся путем выбора позиции из списка.

![](image-17.png)

* **Категория договора** – поле, которое не обязательно для заполнения. Данные вносятся из справочника.
 
* **Куратор** – обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Заполняется автоматически данными из сотрудника.

* **Должность, Подразделение** – поля, которые заполняются автоматически данными Куратора. Поля не редактируемы.

![](image-3.png)

* **Основание договора** – поле, которое не обязательно для заполнения. Данные вносятся из справочника. 

* **Своя организация** – поле заполняется автоматически при открытии нового документа. 

![](image-18.png)

#### 3. Окно “Оплата договора”. 

**Оплата договора** – это согласованный сторонами способ расчетов за выполнение обязательств. 

![](image-19.png)

* Заполнив поля **“Сумма без НДС”** и **“Ставка НДС”** (обязательное для заполнения поле нельзя отключать через статические поля Настройке документов) и **“Сумма с НДС”**, система автоматически рассчитает значения в других полях оплаты.

* **Валюта договора, Валюта платежей, Форма оплаты** – данные выбираются из списка. 

* **Курс валюты** – прописывается вручную. 

* **Сумма в валюте** - заполняется автоматически после выбора Валюты платежей и заполнения Курса валюты.

* Для заполнения условий оплаты, нужно перейти в форму по кнопке **«Условия оплаты»**. По кнопке "Добавить" внести условия. По желанию можно выбрать из справочника шаблон. 

![](image-4.png)

#### 4. Окно "Содержание договора".

В данном окне прописываются условия, на которых достигнуто соглашение сторон.

![](image-20.png)

* **“Предмет договора”** – поле, которое обязательно для заполнения. Оставляя это поле пустым, возможно сохранить его, но для отправки по маршруту необходимы данные в нем. 

* **“Цель заключения”, “Краткое содержание”, “Особые условия”** – поля, которые не обязательны для заполнения. Данные вносятся из вручную. 

#### 5. Окно "Параметры договора".

![](image-21.png)

* **Автопролонгация** означает, что договор будет автоматически продлеваться на определенный срок. Галочка ставится в том случае, если не указана дата окончания действия договора в окне “Регистрация договора” и выбран один из нескольких Статусов: “Утвержден(Подписан)”, “На согласовании”, “Отправлен контрагенту”, “Заключен”.

* **Дата пролонгации** – указывается дата, в которую будет продлеваться договор, не имеющий дату окончания. Дату можно прописать вручную или выбрать при помощи ![](7-1.jpg).

* Если договор рамочный, в поле **Рамочный** нужно выбрать "да", если "нет", выбрать это значение и обязательно заполнить поле "Сумма договора".

***Примечание:*** **рамочный договор** — это договор, открытый для конкретизации. Он определяет только общие или предварительные условия по обязательствам сторон, и стороны конкретизируют эти условия с помощью отдельных договоров, приложений или другим образом.

* Если договор срочный, то ставится признак **Срочный**. В этом случае в договоре должна стоять дата окончания действия (заполнено поле “Действует по” в Окне “Регистрация договора”). 

***Примечание:*** **срочный договор** – это соглашение, в котором определен срок его действия.

* **Проект** – поле, которое не обязательно для заполнения. Данные вносятся путем выбора позиции из списка.

* **Форма** - обязательное для заполнения поле (нельзя отключать через статические поля Настройке документов). Автоматически заполняется данными по умолчанию.

* **Способ закупки, Идентификационный код закупки, Номер аукциона, Номер заявки** – поля, которые не обязательны для заполнения. Данные вносятся вручную. 

* **Причины срочности**. Данные вносятся, в случае, если проставлен признать “Срочный”. 

* **Конфиденциальность** – отмечается, если Стороны договорились, что вся предоставляемая друг другу информация по Договору или в связи с ним считается конфиденциальной.

Конфиденциальные договора будут видимы только тем пользователям, у которых есть соответствующая роль.

* **Приоритет** – необязательное для заполнения поле. Данные выбираются из списка. 

#### 6. Окно "Подразделения и статьи ДДС".

![](image-6.png)

**Подразделение ДДС** (Движение денежных средств) – это структурное подразделение компании, отвечающее за управление денежными потоками. Оно контролирует поступления и выплаты, анализирует ликвидность, прогнозирует остатки средств и обеспечивает финансовую устойчивость предприятия. Подразделение можно выбрать в выпадающем списке.

**Статьи ДДС** - Это элементы классификации денежных потоков в отчетности. Статью ДДС можно выбрать в выпадающем списке.

### Вкладка "Направления"

Источник заключения выбирается из справочника. Направление вносится по кнопке **"Добавить"**. Данная вкладка не обязательна для внесения информации.

![](image-24.png)

### Вкладка "Банковские реквизиты"

Реквизиты своей организации выбирается из справочника. Отображаются те данные, которые внесены в справочнике контрагентов по своей организации. 

Реквизиты контрагента заполняются автоматически при заполнении поля "Контрагент 1 и Контрагент 2" на Основной вкладке.

![](image-7.png)

### Вкладка “Исполнение договора”

В данной вкладке хранится информация о исполнении договора на ту или иную дату. Данная вкладка не обязательна для внесения информации. 

![](image-25.png)

* **Итого по состоянию на** – заполняется текущей датой. По желанию можно изменить. 

* **Сумма договора с доп.соглашениями/без  доп.соглашений** – автоматически заполняемые поля, редактирование которых невозможно.

* **План, факт, остаток** – формируются автоматически, опираясь на ранее внесенные данные .

При нажатии **"Номер"** в табличной части можно перейти в платежное поручение.

### Вкладка “ГОЗ”

**Государственный оборонный заказ** – установленные нормативным правовым актом Правительства Российской Федерации задания на поставки товаров, выполнение работ, оказание услуг для федеральных нужд в целях обеспечения обороны и безопасности Российской Федерации, а также поставки продукции в области военно-технического сотрудничества Российской Федерации с иностранными государствами в соответствии с международными обязательствами Российской Федерации.

![](image-8.png)

!!! Примечание
    Для того, чтобы видеть эту вкладку у вас должна быть добавлена константа **Учет ГОЗ**, значение 1.

* **Гособоронзаказ** - ставится галочка, если поставка товаров, выполнение работ или оказание услуг выполняется для обороны и безопасности страны.

* **Идентификатор госконтракта** - заполняется, если стоит галочка "Гособоронзаказ", проверяется количество символов (25).

* **Номер приказа, номер и дата госконтракта, головной исполнитель госконтракта** – редактируемое поля. Данные вносятся вручную. 

* **Вид договора, способ расчета** – данные берутся из справочника. 

* **Дата приказа, дата подписания акта о сдачи ГОЗ** – выбираются при помощи календаря или вводятся вручную. 

* **Военная приемка, сдача отчета в Мин. обороны** – при выполнении этих пунктов в документе проставляется галочка.
