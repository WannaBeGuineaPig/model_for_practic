# Входящие документы

**Входящий документ** служит для отображения данных по входящей корреспонденции организации. В системе ведется регистрация всех видов входящих документов в электронной форме. Регистрацией и распределением входящей документации в большинстве организаций занимается делопроизводитель, либо работник отдела канцелярии. Далее документ отправляется посредством документооборота для ознакомления и дальнейшей работы сотрудникам в соответствии с утвержденными маршрутами на предприятии. В ходе рассмотрения на документ накладывается резолюция и в зависимости от принятого решения осуществляются дальнейшие действия, например, передача на исполнение сотруднику, отправка на ознакомление группе сотрудников, отклонение документа и т.д.
Из входящего документа могут быть созданы документы «Поручение», «Исходящий документ», «Служебная записка». 

Для каждого типа входящего документов настраивается карточка документа, документам присваивается номер, позволяющий осуществлять быстрый поиск, также возможно настроить справочники, для быстрой подстановки типовых данных. Сами документы приложены к карточке документа в качестве вложенных файлов.

В большинстве случаев обработка входящей документации разделена на три этапа:

1. Регистрация документа

2. Согласование документа (ознакомление)

3. Исполнение


## **Порядок работы с документом**

![Рисунок 4 Регистрация входящего документа](media/screen100.png)

**Регистрация документа**

Инициатор(и регистратор в одном лице):

**1.**  Создает документ;
   
Для создания **нового входящего документа** необходимо в Меню в поисковой строке ввести - «Входящие документы». 

![Picturies](media/menu.png){ width=60% }

Откроется реестр входящих документов за выбранный период.

![Рисунок 2 Реестр входящих документов. Создание нового документа](media/incoming.png)

Кнопка **«Создать»** в панели управления вверху позволяет сформировать новую форму входящего документа.

![alt text](media/image.png)

**2.**  Заполняет необходимые поля: Тип документа, Входящий номер, Входящая дата, Требуется ответ, Срок предоставления ответа, От кого, Кому, Краткое содержание, Содержательная часть. [Побробнее об реквизитах документа](../../Входящий документ/#реквизиты_документасылки).

**3.**  Регистрирует документ в системе по кнопке **«Зарегистрировать»**, если номер и дата не введены вручную, то системой присвоится Регистрационный номер документа и Дата регистрации.
В номер регистрации включаются индексы объектов, например, вид документа, подразделение и тд.
Лучше использовать автоматическую нумерацию системы, так как она позволит избежать неточностей и накладок, возможных при ручном вводе данных.
![Рисунок 4 Регистрация входящего документа](media/screen-3.png)

**4.**  Вкладывает скан-образ документа по кнопке **«Вложения»** на верхней панели инструментов.

**5.** Для согласования или иных действий документ отправляют по маршруту. Для этого регистратор создает маршрут по кнопке **«Маршрут»** на панели инструментов. Если у данного вида документа несколько настроенных маршрутов, то сотрудник выбирает шаблон маршрута из предложенных системой. При необходимости этапы, участники маршрута и виды действия по маршруту могут быть отредактированы. Отправляет документ на следующий этап, нажав кнопку **«Отправить»**

Более подробно работа с маршрутами описана в разделе [Как отредактировать маршрут?](../../Работа с маршрутами/).

![Рисунок 5 Отправка по маршруту для согласования](media/scr4.png)

 В случае ознакомления с документом широкого списка лиц следует сделать рассылку через сервис «Список рассылки». Добавить пользователей и нажать кнопку «Отправить»

Более подробно работа со списками рассылки описана в разделе [Списки рассылки](../../Обработка документов/#список_рассылки).

![Рисунок 6 Создание списка рассылки](media/scr5.png)

**Исполнение входящего документа**

![Рисунок ](media/screen102.png)

Исполнение документов осуществляют специалисты соответствующих структурных подразделений.
На этом этапе исполнителю доступны для редактирования поля "Требуется ответ" , "Срок предоставления ответа", "Ответ предоставлен в срок" и поле "Проект".

При необходимости сотрудник может создать поручение, исходящий документ, служебную записку по кнопке **«Операции«** в верхнем панели документа. При этом созданные документы на основе входящего документа через кнопку **"Операции"** будут видны при нажатии на кнопку **"Связки"**.

![Рисунок 8 Интерфейс создания исходящего документа, поручения и служебной записки из входящего документа](media/screen-4.png)

После подтверждения действий всеми участниками данного этапа, маршрут завершается.

Отчетная форма для входящих документов не предусмотрена.

### **Основные реквизиты входящего документа**: 

Название реквизита | Тип реквизита | Описание
-----------------|---------------|------------
**Регистрационные данные документа** |  | 
Тип документа | Справочник | Заполняется с помощью выпадающего списка. 
Входящий - номер/от | Строка, Дата | Заполняется в ручную, присваивается номер и дата входящего документа.
Рег. № документа/от | Строка, Дата | Запоняется как вручную так или автоматом при нажатии на кнопку "Зарегистрировать" находящуюся рядом.
В ответ на исх. номер/от | Строка, Дата | Заполняется в ручную, присваивается номер для данного документа, который будет указан в исходящем документе, если создать исходящий документ на основе данного документа через кнопку в панели управления "Операции".
Вид доставки | Справочник | Заполняется в ручную через выпадающий список. Назначается способ доставки через "Почту", "Email" или "Нарочно".
Требуется ответ | Флажок| Ставится признак, если обязательно ответное письмо. Заполняется вручную.
Срок предоставления ответа | Дата | Заполняется вручную, если есть признак "Требуется ответ". В системе можно настроить автоматическое проставление даты.
Ответ предоставлен в срок | Дата | Заполняется вручную после получения ответного документа.
**От кого** |  | 
Контрагент | Справочник | Заполняется с помочью выпадающего списка из значений справочника контрагентов.В случае если нет необходимости заводить конрагента в системе заполняется поле под "Контаргент".
Ф.И.О. отправителя | Строка | По умолчанию заполняется значением "Руководитель" от выбранного конрагента. В случае получения документов от частных лиц, заполняется вручную.
Электронная почта | Строка | Заполняется автоматически, если в справочнике контрагентов заполнен электронный адрес, иначе заполняется вручную.
Договор | Справочник | Выбирается из справочника договоров по данному контрагенту. 
Проект| Справочник | Проект, которому адресовано письмо. Заполняется значением из справочника проектов.
**Кому**|  | 
Организация | Справочник | Заполняется автоматически в зависимости на какой организации находится.
Получатель (на чье имя) | Строка | По умолчанию заполняется значением "Руководитель" от выбранной организации.
Краткое содержание | Строка | Краткое содержание документа, заполняется вручную.
Содержательная часть | Строка | Непосредственный текст документа.

 **Реквизиты документа** 

В зависимости от выбранного типа документа в блоке **«Реквизиты документа»** подтянутся те или иные атрибуты. Их необходимо заполнить и нажать кнопку **"Сформировать документ"**.

### Вкладка **Дополнительная** 

В данной форме документа имеется вкладка "Дополнительная" где при необходимости можно заполнить необходимые реквизиты документа, например такие как "Подразделение", "Журнал регистрации", "Проект" и т.д.

![Рисунок](media/screen-2.png)

Название реквизита | Тип реквизита | Описание
-------------------|---------------|------------
Подразделение | Справочник | Выбирается подразделение автора документа из выпадающего списка
Журнал регистрации | Справочник | При создании документа пользователем журнал определяется автоматически в зависимости от типа документа и подразделения пользователя. Не может быть изменено. Настройка журналов регистрации происходит в документе [Журнал регистрации](../Журнал регистрации/index.md).
Срочность | Справочник | Срочность документа. Для данного поля должны быть настроены соответствующий справочник в системе и шаблон маршрута в зависимости от вида срочности.
Срок исполнения | Дата | Срок исполнения по данному письму, заполняется вручную.
С уведомлением | Флажок | Признак, что письмо доставлено с уведомлением, вводится вручную.
Информация об уведомлении | Строка | Дополнительные комментарии к уведомлению. Вводится вручную.
Направление | Справочник | Направление проекта. Заполняется значением из справочника направлений.
